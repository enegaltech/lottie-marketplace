# Changelog

All notable changes to the **lottie-design** skill and the **enegal-marketplace** plugin marketplace are documented here. This project follows [Semantic Versioning](https://semver.org/) (`MAJOR.MINOR.PATCH`).

## [1.0.0] — 2026-04-30

First public release.

### Skill: lottie-design

- 197-entry curated catalog with verified URLs (CDN-reachable, 200 status):
  - 79 entries from useAnimations (CC-BY 4.0)
  - 8 entries from the LottieFiles community CDN
  - 110 entries from the xvrh/lottie-flutter mirror (Lottie Simple License via MIT mirror)
- Per-entry metadata auto-generated by `enrich-catalog.js`: `dominantColor`, `frames`, `duration`, `sizeKb`, `dimensions`.
- Four framework templates: React (`@lottiefiles/dotlottie-react`), React Native (`lottie-react-native` v7+), Flutter (`lottie` v3.3+), Vanilla web (`@lottiefiles/dotlottie-wc`).
- Standardized component interface: `<Component size={n} className?=string />`.
- Auto-install of framework dependency upfront, with package-manager detection (`npm` / `pnpm` / `yarn` / `bun` / `flutter pub`).
- TypeScript vs JavaScript auto-detection for React / RN; emits `.tsx` or `.jsx` accordingly.
- License-aware code generation: emits `// Source: <url> — <license>` header; for CC-BY entries, emits a separate ready-to-paste attribution string.
- CDN mode (default) and local-asset mode (`download-asset.js`) with framework-conventional paths and Flutter `pubspec.yaml` auto-patching.
- Single-animation preview HTML (`generate-preview.js`) with browser auto-open across Windows / macOS / Linux.
- Side-by-side preview gallery (`--gallery` flag) for visual comparison of N candidates.
- Semantic ranking (Step 3 in `SKILL.md`): weighted scoring across intent (0.5), color match (0.2), style/duration (0.15), size (0.15).
- Color-aware match: 17-hue-family color name → hex map, multilingual.
- i18n keyword maps for Turkish, Spanish, German, French, Japanese (≈150 terms total).
- Optional Voyage AI embeddings precompute (`build-embeddings.js`) for offline cosine-sim ranking on large catalogs.
- Catalog scrape pipeline:
  - `scrape-candidates.js` discovers bodymovin JSONs from MIT/Apache GitHub repos with sanity bounds (sizeKb, frames, dimensions).
  - `merge-candidates.js` idempotently folds candidates into `catalog.json`.
